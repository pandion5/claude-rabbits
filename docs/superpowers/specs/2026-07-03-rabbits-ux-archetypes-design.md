## 4. 신규 아키타입 3종 (`archetypes.md` 14종 → 17종)

전부 공통 규칙 상속, 기존과 같은 구조(역할선언·책임·하드제약·출력계약).

### 4.1 요약표

| # | 아키타입 | 기본 subagent_type / 모델 | 핵심 출력 (블록 매핑) |
|---|---------|--------------------------|----------------------|
| 15 | 여정 점검가 (Journey Walker) | `claude` / sonnet | deliverable=시나리오별 주행 결과 + 막힘 지점 목록(위치·증상·심각도), evidence=단계별 실행 기록(명령·스크린샷·출력 발췌) |
| 16 | UX 분석가 (UX Analyst) | `claude` / sonnet | deliverable=발견 목록(위반 원칙·심각도·개선 제안), evidence=발견별 화면/코드 근거 |
| 17 | 디자인 감사가 (Design Auditor) | `claude` / sonnet | deliverable=영역별 정합 판정(일치/불일치 목록), evidence=스크린샷 쌍 + diff 영역 좌표·설명, notes=캡처 환경 조건(뷰포트 등) |

### 4.2 아키타입별 하드 제약

- **여정 점검가**: 시나리오를 **실제 주행**한다(브라우저·CLI 등 작업이 지정한 수단).
  막힘 = 진행 불가·오류뿐 아니라 **"다음에 뭘 해야 할지 모르겠는 지점"** 포함.
  주행 수단이 없으면 BLOCKED(U3). 주행하지 않은 단계를 통과로 보고하지 않는다.
- **UX 분석가**: 발견마다 **위반한 원칙·기준을 인용**한다(U4) — "느낌상 불편"은 발견이
  아니다. 심각도(치명/중요/사소) 분류 의무.
- **디자인 감사가**: 원본(A)·구현(B)을 **동일 뷰포트·동일 조건에서 스크린샷** 떠서
  **픽셀 단위 대조**(U2). 뷰포트는 작업이 지정하고, 지정 없으면 **1280×720 기본**.
  불일치는 영역·좌표로 특정한다. 캡처 불가 환경이면 BLOCKED — 어림 대조로 강등
  금지(U3). 판정만 하고 수정 제안은 영역별 1줄로 제한.

### 4.3 경계 명문화 (해당 템플릿 안, U5)

- **여정 점검가 vs 검증가**: 검증가 = 기능이 동작하나(기술 관찰) / 여정 점검가 =
  사용자가 여정을 통과하나 — **동작해도 사용자가 못 찾으면 막힘이다**.
- **UX 분석가 vs 여정 점검가**: 여정 점검가 = 특정 시나리오 주행(동적 발견) /
  UX 분석가 = 전문 기준 전반 평가.
- **디자인 감사가 vs UX 분석가**: 감사가 = 원본과 **같냐**(정합) / 분석가 = 경험이
  **좋냐**(품질).

## 6. 비목표 (YAGNI)

- **UX팀 신설 / QA팀 편입** — 아키타입만(U1). 팀 조합은 대장 재량 편성으로 충분.
- **픽셀 diff 자동화 도구 지정** — 도구는 워커가 환경에서 가용한 것을 쓴다
  (스크린샷 + 시각 대조). 특정 도구 의존을 프로토콜에 박지 않는다.
- **토큰·레이아웃 수준 대조 모드** — 픽셀 단위로 확정(U2). 완화 모드는 만들지 않는다.
- **SKILL.md 발동 신호 추가** — 팀이 아니므로 요약표 대상 아님.
